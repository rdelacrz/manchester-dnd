# MVP product and policy resolutions

Status: accepted for the private MVP on 2026-07-14. Owner: project maintainer. These
choices intentionally favor a small, testable, privacy-preserving game. A later change
to a durable rule, consent scope, visibility rule, or content choice requires a new
version and an explicit migration plan.

Unless a row says otherwise, the alternatives considered were a broader public or
multiplayer launch and a larger SRD surface. They were rejected because they multiply
authorization, rules, licensing, accessibility, and recovery work before the core game
has been validated. The initial durable versions remain `srd-5.1-cc`,
`content-pack/v1`, and schema version 1. Existing private Slice 1A saves require no
migration; capability validation hides unsupported choices. Rollout is local/private
first, with disabled providers and inspiration off by default.

| ID | Accepted decision | Test and rollout consequence |
| --- | --- | --- |
| Q02 | Support the current and previous major stable Chromium and Firefox releases on Windows, macOS, and Linux desktop; current stable Safari on macOS; and current stable Safari on iOS plus Chrome on Android at 390×844 CSS px or larger. The CI contract is the exact Chromium, Firefox, and WebKit revisions pinned by Playwright. Target WCAG 2.2 AA, English (`en`) only, store UTC instants, and display explicit local time with UTC available. | Browser tests run at desktop 1440×900 and mobile 390×844. Manual evidence covers real Safari/iOS and Chrome/Android because emulation is not a device claim. Unsupported browsers receive usable SSR HTML rather than a blank shell. Review the matrix at each private release. |
| Q03 | Character creation uses the fixed SRD standard array `15, 14, 13, 12, 10, 8`; there is no random ability generation in MVP. | Rust validates one use of each score and records the assignment. No RNG audit or reroll UI is exposed. |
| Q04 | MVP exposes human adventurers; fighter and wizard levels 1–2; soldier and sage backgrounds; simple weapons, longsword, light crossbow, shield, chain mail, leather armor, explorer/scholar packs, spellbook, and arcane focus. The wizard surface is Fire Bolt, Light, Mage Hand, Magic Missile, Shield, and Sleep. The encounter uses the fixed Canal Warden fighter and one original creature, the Soot Wight. Everything else is unavailable rather than partially simulated. | A machine-readable capability report must fail CI if exposed content references any other mechanic. Adding or changing an option requires a new immutable content-pack version and fixtures. |
| Q06 | Use SRD-style unconsciousness at 0 HP, death saves on the affected hero's turn, stabilization after three successes, death after three failures, a natural 1 counting twice, a natural 20 restoring 1 HP, and damage failure rules. MVP campaigns default to a clearly labelled non-terminal `story_recovery` safety policy: defeat ends the encounter, preserves history, restores the hero to 1 HP in exploration, and awards no victory XP. A `rules_as_written` policy may be selected explicitly. | Golden vectors cover every transition. The policy is pinned at campaign creation and never silently changes an existing campaign. |
| Q07 | AI difficulty bands map in Rust to DC 5/10/15/20/25/30 for very easy/easy/moderate/hard/very hard/nearly impossible. The player must confirm before a roll that can cause death, permanent loss, irreversible story commitment, consent-sensitive inspiration, or expenditure of a non-renewable resource. | The model cannot submit a numeric DC. Schema and evaluation tests reject numeric or unrecognized bands and bypassed confirmations. |
| Q08 | Ship deterministic fake and disabled providers for both modalities plus the existing operator-configured OpenAI-compatible adapter. No real provider is approved for private inspiration by this decision. An operator must record provider/plan, retention, training use, processing region, moderation, deletion, and private-input eligibility before enabling it; otherwise only minimized non-private facts may be sent. | Provider profiles fail closed, are restart-applied, and retain a non-secret fingerprint. CI uses only fake/disabled providers. Private inspiration remains ineligible for provider calls until an approval record exists. |
| Q09 | Images are manual/on-demand only. Default cap: 3 requests per campaign per rolling 24 hours, 10 per campaign lifetime, one running job per campaign, and an operator-configured monetary hard cap defaulting to zero for non-fake providers. | Budget is checked transactionally before enqueue. The UI shows remaining requests and a cost estimate; exceeding any cap returns a stable non-retryable code. |
| Q10 | Narration permits at most two presentation regenerations per committed turn; images permit one replacement request. Mechanics, rolls, and source selection never rerun. All versions remain owner-visible for 30 days, selected versions remain with the campaign, ordinary exports include selected versions only, and canonical exports include version metadata but no rejected body unless explicitly requested. | Regeneration keys include the committed turn and policy fingerprint. Tests prove identical mechanical state before and after retries and enforce expiry/selection rules. |
| Q11 | Private testing is adults-only (18+). Prohibit sexual content involving minors, real-person likeness, current personal crisis, and unconsented health, trauma, sexual, criminal, financial, employment, address/contact, or relationship secrets. Threats of imminent harm trigger generation shutdown and the operator escalation path; the game is not an emergency service. | Safety setup is required before campaign play. Sources and outputs are scanned conservatively, uncertain material is quarantined, and the global kill switch is rehearsed before external testing. |
| Q12 | The local campaign operator creates opaque participant IDs and verifies every grant or revocation out of band using a signed confirmation or timestamped two-channel acknowledgement. Only that participant may grant consent; either the participant or operator may revoke it immediately. The operator may not self-attest for another person. | The protected mapping is separate from gameplay data. UI/API responses reveal only the current user's grants and aggregate eligibility, never another participant's identity or consent body. |
| Q13 | Active campaigns, created heroes, selected artifacts, and immutable mechanical audits live until owner deletion. Incomplete hero-creation drafts expire after 7 days, remain available only for recovery/audit for 30 additional days, then are deleted. Archives live until restored/deleted. Failed generation metadata expires after 7 days; restricted diagnostics after 14 days; unselected presentation versions after 30 days; prepared exports after 24 hours; revoked raw sources are removed from active storage immediately; encrypted backups expire after 30 days. Deletion tombstones containing only opaque IDs/digests may remain 35 days to carry deletion through backups. Full prompts are not retained by default. | Scheduled cleanup is idempotent and audited with opaque IDs. Exact expiry and cleanup-boundary tests cover creation drafts; export/delete and backup-expiry drills verify the wider policy. Legal hold is not an MVP feature. |
| Q14 | Campaigns and recaps are private to the authenticated owner. MVP has no public links, discoverability, or direct canonical-document sharing. A public/redacted projection is post-MVP. | Every route, artifact, history page, and export performs object authorization. Sharing fields and public routes are absent from v1 schemas. |
| Q15 | The private MVP grants no public code, original-content, or pack license; all rights are reserved by their respective owners and outside contributions are closed. This is a distribution constraint, not an open-source license. A public repository, third-party pack intake, or distribution is blocked until the maintainer records explicit code/content licenses and contribution terms. | Release tooling labels artifacts `private-evaluation-only` and refuses a public release profile. Third-party CC material retains its own attribution and terms. |
| Q16 | `Manchester Arcana` remains a private working title with original visual presentation. No domain acquisition, public marketing, or claim of trademark rights is authorized. Public release is blocked pending a maintainer/legal name, domain, and trademark clearance. | UI/legal text calls it a working title and avoids protected product identity, logos, settings, trade dress, and endorsement language. |
| Q17 | Generated NPCs, locations, or prose are never promoted automatically. Promotion requires a human rewrite, rights/likeness/safety review, provenance entry, capability validation, and a new immutable pack version approved by the maintainer. | Creator tooling and promotion APIs are out of MVP; pack validation rejects generated entries without the review record. |
| Q18 | Collect operational telemetry only: bounded request/turn/job outcome counts, latency, resource health, and denial/fallback counts. Do not capture behavioral funnels, campaign text, prompts, private sources, identifiers, dice seeds, or generated bodies. | Metric labels use bounded enums and no user/campaign IDs. Canary tests scan logs, metrics, responses, and artifacts. Any behavioral analytics require a new opt-in decision. |
| Q19 | Players always see dice, kept values, modifiers, DC/AC, purpose, outcome, algorithm ID, and cursor range. Raw seed material remains encrypted/server-only; audits retain an opaque seed reference, the pinned algorithm (`chacha20-v1`), and cursor before/after. | Rendering and replay consume stored `RollRecord`s. Retries, reload, history, exports, and corrections cannot advance the cursor. Multiplayer commitment/reveal is post-MVP. |
| Q20 | Every schema/ruleset/content version used by a privately released campaign remains readable and exportable until an explicit end-of-support record supplies a migrator or lossless canonical export, at least two release notices, and 90 days' notice. Unsupported future semantics open read-only rather than being silently reinterpreted. | CI loads all released fixtures and rejects removal of referenced packs/migrators. There is no in-place semantic rewrite of old audits. |

## External-release holds

The working private game does not imply permission to distribute it publicly. Public
release remains blocked on Q15 license selection, Q16 legal clearance, a real-provider
Q08 privacy review for every enabled profile, and the Slice 7 security, privacy,
accessibility, recovery, and provenance evidence. These holds fail closed in release
configuration and are not bypassed by a successful local build.
